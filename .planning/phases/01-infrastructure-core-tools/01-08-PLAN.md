<objective>
Fix UAT Test 15 gap: pinned-tool drag-to-reorder is completely dead. The drag never starts
because .onDrag is layered on a SwiftUI Button, whose press gesture preempts the drag on macOS,
so no NSItemProvider is vended, PinnedToolDropDelegate.performDrop never fires, and
prefs.movePinnedTool is never called. A latent secondary defect, an off-by-one in the
performDrop destination math, would surface as "reorder lands one slot too far" once the gesture
is fixed.

Purpose: Restore drag-to-reorder (INFRA-11) so the pinned bar matches its UAT spec.
Output: A working drag source decoupled from the Button, corrected destination index math, and a
regression test pinning the move-index semantics.
</objective>

From Core/Services/PreferencesStore.swift:
- var pinnedToolIds: [String] { get set }   // persisted to UserDefaults
- func movePinnedTool(from source: IndexSet, to destination: Int)  // internally ids.move(fromOffsets:toOffset:)

Key fact about Array.move(fromOffsets:toOffset:): toOffset uses the insert-before-index in the
ORIGINAL array convention, the SAME convention SwiftUI List.onMove hands its caller. A raw
firstIndex(of:) destination result is already in that convention. The current code adds +1 for
forward moves, double-compensating and landing the tool one slot past the target.
</interfaces>
