  <behavior>
    - Parse "255" (dec, 8-bit unsigned) → pattern 0xFF; emit bin "11111111", oct "377", hex "FF".
    - 8-bit signed: pattern 0xFF renders DEC as "-1"; pattern 0x80 renders "-128"; 0x7F renders "127".
    - 16/32/64-bit signed boundaries: Int16 min/max, Int32 min/max, Int64 min (-9223372036854775808) / max render correctly.
    - Parse "FF" (hex) → all bases consistent; parse "777" (oct) and "0b101" tolerant of prefixes per chosen convention.
    - Overflow: parsing a value exceeding the active width masks to width AND returns an overflow flag (e.g. 256 in 8-bit → 0 + overflow=true), never crashes.
    - Toggle bit: XOR bit index i flips that bit and re-derives all bases.
    - Garbage: invalid digit for base (e.g. "2" in binary, "G" in hex), empty, lone "-" → graceful failure/last-good, never crash.
  </behavior>
  <action>Create `enum NumberBaseTransformer` (`import Foundation` only). Canonical representation: a `UInt64` bit pattern + `width` (8/16/32/64) + `signed` flag. Implement `parse(_ text: String, base: NumberBase, width: BitWidth) -> Result<ParseResult, TransformError>` where ParseResult carries the masked `UInt64` pattern and an `overflow: Bool`. Implement emitters: `binary(pattern:width:)`, `octal(...)`, `decimal(pattern:width:signed:)`, `hex(...)` using `String(value, radix:, uppercase:)`. Decimal signed rendering: mask to active width, and if the top bit of the active width is set, value = `pattern - 2^width` (so 8-bit 0xFF → -1) — implement via sign-extension using the width's `Int(bitPattern:)` on the masked value, NOT raw 64-bit. Masking: `pattern & ((width==64) ? .max : ((1 << width) - 1))` (guard the 64-bit `1<<64` UB — special-case width 64). Overflow: if the parsed magnitude exceeds the width's representable range, mask + set overflow=true (NUM-03 — never reject silently, never crash). `toggleBit(pattern:index:) -> UInt64` via `pattern ^ (1 << index)`. Define `enum NumberBase { case bin, oct, dec, hex }` and `enum BitWidth: Int { case w8=8, w16=16, w32=32, w64=64 }`. All total/Result-returning; never force-unwrap (INFRA-17). Write `FlintTests/NumberBaseTransformerTests.swift` covering the FULL matrix: each width × signed/unsigned × {0, max, min, -1, overflow-by-one, empty, invalid-digit-for-base}. Bake the Int8 (-128/127) and Int64 (min/max) boundaries explicitly.</action>
