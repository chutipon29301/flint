<objective>
Build the pure, testable core of the Image Compressor: `ImageCompressTransformer`. It performs the ImageIO CGImageSource→CGImageDestination round-trip re-encode (same format in = same format out), computes the collision-safe `-compressed` destination path beside the original, and reports before/after byte counts — all without ever throwing or crashing across the UI boundary (INFRA-17). This is the linchpin file and the only one with a unit-test acceptance criterion.

Purpose: Isolate all ImageIO + filesystem-write logic in one pure enum so it can be unit-tested on malformed input before any UI exists. Net-new surface — there is no in-codebase analog for either the ImageIO round-trip or the beside-original write, so this plan implements RESEARCH.md Pattern 1 + Pattern 2 exactly.

Output: `Tools/ImageCompress/ImageCompressTransformer.swift`, `FlintTests/ImageCompressTransformerTests.swift`, both added to their Xcode targets.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dropped file URL → Transformer | Untrusted file content/path crosses into ImageIO decode + a filesystem write |
| Transformer → filesystem (write beside original) | First write-to-disk surface in the app; data-integrity boundary (must not destroy the source) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Denial of Service | `compress()` on corrupt/truncated/non-image/0-byte input | mitigate | Every ImageIO call guard-gated to a typed `.failure`; function never throws/crashes (INFRA-17). Proven by the corrupt-input + 0-byte unit tests in Task 2. |
| T-05-02 | Tampering / data loss | `disambiguatedCompressedURL` + `Finalize` | mitigate | Destination derives from the dropped file's own URL with a `-compressed` suffix and a `fileExists` collision loop — the original is never the destination, never overwritten (D-08). Partial writes on `Finalize` failure are removed. Best-effort TOCTOU window accepted for a single-user, non-sandboxed desktop tool (RESEARCH A3). |
| T-05-03 | Tampering / path redirection | output path math | accept | Path is built purely from the dropped URL's own `deletingLastPathComponent`/`lastPathComponent`/`pathExtension`; no user-supplied filename string is concatenated, so a crafted name cannot redirect the write outside the source's own directory. No traversal vector introduced. |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No package installs this phase — zero new external dependencies (D-03; RESEARCH Package Legitimacy Audit "Not applicable"). Legitimacy gate not triggered. |
</threat_model>
