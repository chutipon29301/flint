<objective>
Build the two pure-Swift engine pieces that make real PNG compression possible: a color quantizer that reduces a truecolor RGBA image to a <=256-color palette, and an indexed-color PNG encoder that writes that palette as a color-type-3 PNG (which Apple's CGImageDestination cannot emit).

This is the root-cause fix for UAT Test 8: Flint barely shrinks photographic PNGs because the transformer re-encodes them as truecolor RGBA. The reference online tool shrank a 7.27 MB PNG to 1.35 MB by converting 8-bit RGBA truecolor to 8-bit colormap (256-color palette) — exactly what this engine produces.

Both pieces are pure Swift with ZERO external dependencies (locked decision: libimagequant/pngquant are GPL/LGPL and would block App Store v2 sandbox). Both have well-defined input/output, so this plan is TDD-friendly.

Purpose: Quantization + indexed-PNG writing are the only path to pngquant-class savings; ImageIO has no palette API.
Output: PNGColorQuantizer.swift, IndexedPNGEncoder.swift, PNGQuantizationTests.swift. Wiring into the transformer happens in 05-05.
</objective>

  <name>Task 1: IndexedPNGEncoder — pure-Swift color-type-3 PNG writer</name>
  <files>Tools/ImageCompress/IndexedPNGEncoder.swift, FlintTests/PNGQuantizationTests.swift</files>
  <behavior>
    - Given a width, height, a palette of <=256 RGB triples, an alpha array aligned to the palette (or nil for fully opaque), and a per-pixel index array of length width*height, `encode` returns Data that is a valid PNG.
    - The returned Data starts with the 8-byte PNG signature 0x89 'P' 'N' 'G' 0x0D 0x0A 0x1A 0x0A.
    - The Data, written to disk, is openable by ImageIO: `CGImageSourceCreateWithURL` returns non-nil, `CGImageSourceGetType` == "public.png", and the decoded image's width/height match the inputs.
    - When an alpha array contains any value < 255, a tRNS chunk is emitted (verify the file decodes with an alpha channel / the bytes contain the ASCII "tRNS").
    - Edge: width==0 or height==0, or palette empty, or indices length mismatch -> returns nil (no crash, INFRA-17).
    - A 64x64 quantized image encodes to fewer bytes than the same image re-encoded as truecolor RGBA PNG via CGImageDestination (proves the size win at the encoder level).
  </behavior>
  <action>
    Create `Tools/ImageCompress/IndexedPNGEncoder.swift`. Implement an enum `IndexedPNGEncoder` with a single static func, e.g. `encode(width:height:palette:alpha:indices:) -> Data?`.

    PNG structure to emit (color type 3, indexed-color, bit depth 8):
    1. 8-byte PNG signature.
    2. IHDR chunk: width (UInt32 BE), height (UInt32 BE), bit depth = 8, color type = 3, compression = 0, filter = 0, interlace = 0.
    3. PLTE chunk: 3 bytes (R,G,B) per palette entry, in palette order.
    4. tRNS chunk: ONLY if any palette entry has alpha < 255. One alpha byte per palette entry up to and including the last non-opaque entry (PNG allows omitting trailing 255s, but emitting all is also valid — keep it simple: emit `alpha.count` bytes when alpha is present and any entry < 255).
    5. IDAT chunk(s): build the raw scanline stream as, for each row, a leading filter-type byte 0x00 (None) followed by one index byte per pixel; then zlib-deflate the whole stream with the Compression framework (`compression_encode_buffer`, algorithm `COMPRESSION_ZLIB`). Note `COMPRESSION_ZLIB` produces the raw DEFLATE/zlib stream PNG expects — confirm the output begins with a zlib header (0x78). If the platform variant omits the 2-byte zlib header, prepend a fixed zlib header (0x78 0x01) and append the Adler-32 checksum; implement Adler-32 in-file (simple, ~12 lines) as a fallback. Prefer the framework's zlib output if it already includes header+checksum.
    6. IEND chunk (empty).

    Each chunk on the wire is: length (UInt32 BE of data bytes) + 4-byte ASCII type + data + CRC-32 (UInt32 BE) computed over type+data. Implement the standard PNG CRC-32 (polynomial 0xEDB88320) as a private helper with a lazily-built 256-entry table. Do NOT use zlib's crc32 from the Compression import path — implement the table-driven CRC locally so there is no bridging-header dependency.

    Use big-endian serialization helpers for all UInt32 length/CRC fields (PNG is network byte order). Validate inputs up front: guard width>0, height>0, !palette.isEmpty, palette.count<=256, indices.count == width*height, and every index < palette.count; return nil on any violation (INFRA-17, no crash).

    In the SAME file's accompanying test file `FlintTests/PNGQuantizationTests.swift`, create the `@Suite("PNGQuantization")` with the IndexedPNGEncoder tests described in <behavior>. Mirror the existing test style in ImageCompressTransformerTests.swift: `import Testing`, `@testable import Flint`, temp-dir helper, synthesise inputs in code (no binary assets), assert via `#expect`. Write the encoder output to a temp file and re-open it with CGImageSourceCreateWithURL to prove validity. (PNGColorQuantizer tests are added in Task 2 to this same suite/file.)

  <name>Task 2: PNGColorQuantizer — median-cut RGBA -> palette + index map</name>
  <files>Tools/ImageCompress/PNGColorQuantizer.swift, FlintTests/PNGQuantizationTests.swift</files>
  <behavior>
    - Given a CGImage (or a decoded RGBA8 pixel buffer + width/height), `quantize(maxColors:)` returns a value with: `palette: [(r,g,b)]` (count <= maxColors, <= 256), `alpha: [UInt8]` aligned to palette, and `indices: [UInt8]` of length width*height where each entry < palette.count.
    - An image with <= maxColors distinct colors yields a palette of exactly the distinct-color count and a lossless index map (decoding indices through the palette reproduces the source pixels exactly).
    - A photographic gradient image (e.g. 64x64 with hundreds of distinct colors) yields a palette of exactly maxColors (e.g. 256) and an index map covering all pixels; reconstructing through the palette is visually close (assert max per-channel error below a generous threshold, e.g. each reconstructed channel within a tolerance — exact value chosen so a smooth gradient passes).
    - Alpha is preserved: a buffer with transparent pixels yields palette entries whose alpha array carries the transparency; reconstructed alpha matches within tolerance.
    - Edge: 1x1 image, fully-uniform image (1 color), width/height 0 -> returns nil or an empty-safe result without crashing (INFRA-17).
  </behavior>
  <action>
    Create `Tools/ImageCompress/PNGColorQuantizer.swift`. Implement an enum `PNGColorQuantizer` exposing a struct `QuantizedImage { let width: Int; let height: Int; let palette: [(UInt8,UInt8,UInt8)]; let alpha: [UInt8]; let indices: [UInt8] }` and a static func `quantize(cgImage: CGImage, maxColors: Int = 256) -> QuantizedImage?`.

    Steps:
    1. Decode the CGImage into a tightly-packed RGBA8 byte buffer: create a CGContext with `CGColorSpaceCreateDeviceRGB`, `bitsPerComponent: 8`, `bytesPerRow: width*4`, premultipliedLast or last alpha — match the existing helper convention in ImageCompressTransformerTests.swift (CGImageAlphaInfo.premultipliedLast is fine; document the choice). Guard width>0/height>0 and a successful context/draw; return nil otherwise (INFRA-17).
    2. Run median-cut quantization on the unique pixels (treat RGB for the cut; carry the average alpha per resulting box). Algorithm: start with one box containing all pixels; repeatedly split the box with the largest color-range axis at its median along that axis until you have `maxColors` boxes or no box is splittable. The representative color of a box is the per-channel average of its pixels. This is the classic median-cut described in the locked decision (median-cut or octree — choose median-cut). Keep it readable, not micro-optimized; a 64x64 test image must quantize in well under a second.
    3. Build the palette from box representatives; build a fast nearest-palette lookup (a simple linear nearest-color search in RGB is acceptable for <=256 entries; optionally cache results in a dictionary keyed by packed RGBA to avoid recomputing for repeated pixels) and produce the per-pixel `indices` array.
    4. Populate `alpha` aligned to palette order (box average alpha, clamped 0...255).
    5. Return `QuantizedImage`. Return nil on any guard failure.

    Add the PNGColorQuantizer tests from <behavior> to the SAME `FlintTests/PNGQuantizationTests.swift` suite created in Task 1. For the "round-trips a low-color image losslessly" test, synthesise a CGImage with a known small set of colors and assert palette.count == distinct count and reconstructed pixels == source. For the photographic case, synthesise a smooth gradient CGImage and assert palette.count == 256 and a generous reconstruction tolerance. Add an end-to-end test that pipes `quantize` output straight into `IndexedPNGEncoder.encode` and asserts the resulting Data opens via ImageIO and is smaller than a truecolor re-encode of the same gradient.
  </action>
