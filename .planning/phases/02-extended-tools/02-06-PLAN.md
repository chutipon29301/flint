  </read_first>
  <behavior>
    - Identical inputs → DiffResult with all lines `.unchanged`, no diffs.
    - Pure insert: original "a\nb" vs changed "a\nb\nc" → one added line "c".
    - Pure delete: original "a\nb\nc" vs changed "a\nb" → one removed line "c".
    - Modified line → paired remove+insert at the same position, with word-level segments from SwiftDiff (e.g. "the cat" vs "the dog" → equal "the ", delete "cat", insert "dog").
    - ignoreWhitespace: "a b" vs "a  b" → no diff when ignoreWhitespace true; differ when false.
    - ignoreCase: "ABC" vs "abc" → no diff when ignoreCase true; differ when false.
    - Unified patch: emits `@@ -l,s +l,s @@` hunks with ` `/`-`/`+` line prefixes for a known before/after.
    - Empty inputs (both empty, one empty) → graceful (no crash, INFRA-17).
  </behavior>
  <action>Create `enum TextDiffTransformer` (`import Foundation` only — the vendored SwiftDiff is in-target). Implement `static func diff(original: String, changed: String, ignoreWhitespace: Bool, ignoreCase: Bool) -> DiffResult`. Define `struct DiffResult` containing an ordered `[DiffLine]` (each with kind `.added/.removed/.unchanged`, original/new line numbers, the raw display text, and optional word-level `[WordSegment]` for modified lines) plus a computed `unifiedPatch: String`. Line-level: split both inputs into `[String]` by line, apply ignore-ws/ignore-case as NORMALIZATION used ONLY for comparison (preserve original text for display), then `changedLines.difference(from: originalLines)` (CollectionDifference, Myers). Pair a `.remove` immediately followed by an `.insert` at the same logical position as a MODIFIED line and compute word-level segments by calling the vendored `diff(text1: oldLine, text2: newLine)` → `[Diff]` of `.equal/.insert/.delete`. Implement `unifiedPatch` emitting standard unified-diff (`--- Original`/`+++ Changed` header, `@@ -l,s +l,s @@` hunks, ` `/`-`/`+` prefixes). All total/Result-safe; never force-unwrap (INFRA-17). Write `FlintTests/TextDiffTransformerTests.swift` (XCTest) covering every behavior above, including a vendored-SwiftDiff word-level assertion and the unified-patch output.</action>
