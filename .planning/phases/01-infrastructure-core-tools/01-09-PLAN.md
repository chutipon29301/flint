  <name>Task 1: Create the shared ToolShortcutActions protocol and view modifier</name>
  <files>UI/Components/ToolShortcutActions.swift</files>
  <action>
    Create a single shared contract so all 7 tools subscribe consistently (debug session
    recommendation: centralize rather than copy-paste 7 handlers).

    Define a protocol `ToolShortcutActions` with two requirements (MainActor):
    - func primaryOutput() -> String?   // the active tool's copy-output value; nil/empty means nothing to copy
    - func clearInput()                  // clears the tool's input field(s)

    Define a ViewModifier (and a `.toolShortcuts(_ actions:)` View extension) that, given a value
    conforming to ToolShortcutActions, attaches:
    - .onReceive(NotificationCenter.default.publisher(for: .copyOutput)) { _ in ... } that reads
      actions.primaryOutput(); if it is non-nil and non-empty, writes it to NSPasteboard.general
      (clearContents() then setString(_, forType: .string)). Empty/nil => harmless no-op (never
      crash, INFRA-17 spirit).
    - .onReceive(NotificationCenter.default.publisher(for: .clearInput)) { _ in actions.clearInput() }

    Mark the modifier @MainActor (it touches @Observable view models on the main actor). Import
    SwiftUI and AppKit. Reference .copyOutput / .clearInput which are declared in
    MenuBarPopoverView.swift — do not redeclare them. Keep this file free of any tool-specific or
    GRDB knowledge; it only knows the protocol.

    Note for executor: the value passed in will be the @Bindable view model. Since @Observable
    classes are reference types, capture it directly in the modifier; do not require Equatable.
  </action>
