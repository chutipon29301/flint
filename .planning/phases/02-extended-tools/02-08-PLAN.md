<objective>
Close the MD-02 editor-highlighting gap found in 02-VERIFICATION.md. The requirement is "Editor highlights Markdown syntax; preview highlights code blocks" — preview-side code-block highlighting was delivered in 02-04, but the editor used a plain monospace SyntaxEditorView with no Markdown-syntax pass. This plan adds an opt-in, attribute-only Markdown syntax highlight to SyntaxEditorView and turns it on for the Markdown editor only.

Purpose: MD-02 (D-10). Scope is deliberately minimal — a lightweight regex/scan-based attribute pass over the visible text, NOT a full Markdown parser (the preview already uses swift-markdown for that). The landmine is the Pitfall #5 re-render guard: attribute changes must not retrigger the text-change cycle.
Output: the Markdown editor visibly distinguishes heading markers, bold/italic emphasis markers, inline code, and link syntax as the user types.
</objective>
